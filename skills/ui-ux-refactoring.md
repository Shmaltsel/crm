# SKILL: Редизайн та UI/UX рефакторинг (Tailwind + Framer Motion)

**Коли використовувати**: користувач просить оновити дизайн, переробити верстку під мобільні пристрої (mobile-first), додати анімації або переписати існуючі компоненти інтерфейсу.

## Кроки

1. **Mobile-First підхід**
   - Уникати синдрому "нескінченного сувою". Використовувати вкладки (через `navTabs.ts`) або Bottom Sheets (`MoreSheet.tsx`) для приховування другорядного контенту.
   - Жодних вкладених вкладок (tabs inside tabs) на мобільному. 
   - Кнопки дій робити достатньо великими для тачу (мін. висота 44px).

2. **Анімації (framer-motion)**
   - Максимально перевикористовувати готові примітиви з `src/lib/motion.ts`.
   - Для жестів обов'язково використовувати `dragConstraints` та обробляти `onDragEnd` (наприклад, свайп між вкладками).
   - Враховувати accessibility: жести повинні залишатися функціональними незалежно від налаштувань анімації пристрою.

3. **Стилізація (Tailwind CSS)**
   - Використовувати виключно семантичні токени проєкту (наприклад, `bg-surface`, `bg-brand`, `text-content-secondary`, `rounded-card`, `rounded-pill`).
   - Хардкодити стандартні кольори Tailwind (`slate-`, `blue-` тощо) суворо заборонено.
   - Забезпечити правильний z-index контекст для `AnimatePresence` та фіксованих елементів (`BottomNavigationBar`, `MobileTopNav`).

4. **Empty States та Fallbacks**
   - Замість нейтрального приховування блоку через `{arr.length > 0 && <Block/>}`, завжди передбачати позитивний або інформативний empty state.
