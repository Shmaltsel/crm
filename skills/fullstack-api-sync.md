# SKILL: Fullstack API Sync (Синхронізація даних між BE та FE)

**Коли використовувати**: змінено Prisma-модель, додано нове поле в існуючий DTO на бекенді, або змінено endpoint, що вимагає оновлення фронтенду.

## Кроки (Data Flow Pipeline)

1. **Backend DTO & Controller**
   - Переконатися, що нові поля додані в DTO з правильними декораторами `class-validator`.
   - Якщо це число, обов'язково додати `@Type(() => Number)`.
   - Якщо повертається `Decimal`, переконатися, що воно каститься в `Number` для JSON-відповіді.

2. **Frontend Types (`src/types/index.ts`)**
   - Знайти відповідний інтерфейс (напр., `School`, `EventReport`) і додати/змінити поле, щоб воно точно відповідало Swagger/BE DTO.

3. **React Query Hooks (`src/hooks/use*.ts`)**
   - Перевірити, чи не потрібно оновити параметри мутації або query keys.
   - Переконатися, що `invalidateQueries` зачіпає оновлені дані.

4. **Frontend UI Components**
   - Додати нове поле у відповідну форму (`Input`, `Select`).
   - Вивести нові дані в таблиці або картці.
   - Перевірити fallback (що показувати, якщо нове поле прийшло як `null`).
